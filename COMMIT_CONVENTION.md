# Commit Convention

## 커밋 메시지 형식

```
<type>: <description>
```

## 타입 목록

- feat : 새 기능 추가
- fix : 버그 수정
- docs : 문서 변경
- style : 코드 포맷/세미콜론 등 (기능 변경 없음)
- refactor : 리팩토링 (기능 변경 없음)
- chore : 빌드/설정/잡일/파일 정리
- test : 테스트 추가/수정
- ci : CI/CD 설정
- perf : 성능 개선

## 규칙

- 타입은 영문 소문자
- 설명은 한글 허용
- 본문은 선택 (필요 시 빈 줄 후 작성)

## 예시

```
feat: 로그인 페이지 구현
fix: 비밀번호 validation 오류 수정
docs: README 업데이트
chore: 불필요 파일 제거
refactor: 설문 컴포넌트 구조 개선
```
