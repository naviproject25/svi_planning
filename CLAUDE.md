# 프로젝트 규칙

## Common 폴더 참조 규칙
- 기획 작업 요청 시 → Common/PLANNING.md 참조
- 커밋 작업 요청 시 → Common/COMMIT_CONVENTION.md 참조
- 프로젝트 컨텍스트 필요 시 → Common/SYSTEM_PROMPT.md 참조
- UI 구현 시 → Common/UI_RULES.md 참조

## 기본 규칙
- 이모지 사용 금지
- 결론 먼저 작성
- 관련 파일 수정 시 연관된 모든 파일 함께 수정
- 실서비스 품질 유지 (임시 텍스트 금지)

## 검증 규칙 (엑셀 수식 관련 작업 시 필수)
- 수치 계산뿐 아니라, 해당 수치를 사용하는 텍스트 생성 로직(Result.tsx 등)까지 반드시 대조할 것
- 엑셀 셀 주소와 수식을 명시하고, 코드의 어떤 라인이 대응하는지 1:1 매핑 결과를 보고할 것
- handleExcelFill 등 하드코딩 데이터는 엑셀 원본 셀 값과 직접 비교 후 보고할 것
- "완료"라고 말하기 전에 입력→계산→출력 전체 흐름을 추적할 것 (중간만 검증 금지)
- 수정 후 반드시 `npm run test` 실행하여 엑셀 기대값 테스트 통과를 확인할 것
